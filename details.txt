You are building the next phase of a SaaS product called Fixlytics (website audit + fix tool).

The core audit system is already complete.

Your task is to implement:

* Authentication (signup/login)
* Payment system
* User dashboard
* Subscription logic

Focus on simplicity, scalability, and fast launch.

---

# 🎯 OBJECTIVE

Turn the product into a monetizable SaaS where users can:

* Create accounts
* Run audits
* Save reports
* Upgrade to paid plans
* Access a dashboard

---

# 1. AUTHENTICATION SYSTEM

## Features:

* Signup (email + password)
* Login
* Logout
* Protected routes

---

## Requirements:

### Signup:

* Email
* Password
* Store hashed password (bcrypt)

### Login:

* Validate credentials
* Return JWT token

### Session:

* Store JWT in localStorage or cookies
* Protect dashboard routes

---

## Database (users collection):

Fields:

* id
* email
* password (hashed)
* plan (free / pro)
* auditsUsed (number)
* createdAt

---

# 2. FREE PLAN LIMIT SYSTEM

## Logic:

Free users:

* Max 2 audits per day

---

## Implementation:

* Track auditsUsed per user
* Reset daily OR track timestamps

---

## Behavior:

If limit reached:
Show:
“Free limit reached. Upgrade to continue.”

---

# 3. PAYMENT SYSTEM (STRIPE)

## Use Stripe (recommended)

---

## Features:

* Monthly subscription ($9–$19)
* Checkout page
* Webhook handling

---

## Flow:

1. User clicks “Upgrade”
2. Redirect to Stripe Checkout
3. User pays
4. Stripe webhook updates user plan → “pro”

---

## Backend:

### Create endpoint:

POST /api/create-checkout-session

### Webhook:

* Listen for:

  * checkout.session.completed
* Update user.plan = "pro"
